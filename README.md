# SwiftDeploy

A declarative deployment CLI that generates your entire stack from a single `manifest.yaml`.

## Project Structure

```
project/
├── manifest.yaml            ← ONLY file you edit manually
├── swiftdeploy              ← CLI executable
├── app/
│   ├── main.py              ← Python HTTP API service
│   ├── Dockerfile           ← Single-stage Alpine build
│   └── requirements.txt
├── templates/
│   ├── nginx.conf.tmpl      ← Nginx config template
│   └── docker-compose.yml.tmpl
├── nginx.conf               ← GENERATED by init (do not edit)
├── docker-compose.yml       ← GENERATED by init (do not edit)
└── README.md
```

## Prerequisites

- Docker + Docker Compose v2
- Python 3.8+ with PyYAML: `pip install pyyaml`

## Setup

```bash
# 1. Clone the repo
git clone https://github.com/AnitaAliCloud/hng4-devops
cd hng4-devops

# 2. Make CLI executable
chmod +x swiftdeploy

# 3. Build the Docker image
docker build -t anitaali-swiftdeploy:latest ./app
```

## Subcommand Walkthrough

### `init` — Generate config files
```bash
./swiftdeploy init
```
Reads `manifest.yaml` → generates `nginx.conf` and `docker-compose.yml` from templates.
The grader will delete generated files and re-run this to verify they regenerate correctly.

### `validate` — Pre-flight checks
```bash
./swiftdeploy validate
```
Runs 5 checks before touching Docker:
1. `manifest.yaml` exists and is valid YAML
2. All required fields are present and non-empty
3. Docker image exists locally
4. Nginx port is not already bound on host
5. `nginx.conf` is syntactically valid (via `nginx -t` in a container)

Exits with code `1` if any check fails.

### `deploy` — Start the stack
```bash
./swiftdeploy deploy
```
Runs `init` → `validate` → `docker compose up -d` → polls `/healthz` until healthy (60s timeout).

### `promote` — Switch modes
```bash
./swiftdeploy promote canary   # Switch to canary mode
./swiftdeploy promote stable   # Switch back to stable
```
- Updates `mode` field in `manifest.yaml` in-place
- Regenerates `docker-compose.yml` with new `MODE` env var
- Restarts **only** the app container (Nginx keeps running)
- Confirms new mode is active by calling `/healthz`

### `teardown` — Shut everything down
```bash
./swiftdeploy teardown           # Stop containers + remove network/volumes
./swiftdeploy teardown --clean   # Also delete generated nginx.conf + docker-compose.yml
```

## API Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/` | GET | Welcome message with mode, version, timestamp |
| `/healthz` | GET | Liveness check, returns uptime in seconds |
| `/chaos` | POST | Inject chaos (canary mode only) |

### Chaos Examples
```bash
# Slow mode: sleep 5s before every response
curl -X POST http://localhost:8080/chaos \
  -H "Content-Type: application/json" \
  -d '{"mode": "slow", "duration": 5}'

# Error mode: 50% of requests return HTTP 500
curl -X POST http://localhost:8080/chaos \
  -H "Content-Type: application/json" \
  -d '{"mode": "error", "rate": 0.5}'

# Recover: clear all chaos
curl -X POST http://localhost:8080/chaos \
  -H "Content-Type: application/json" \
  -d '{"mode": "recover"}'
```

## Nginx Access Log Format
```
$time_iso8601 | $status | ${request_time}s | $upstream_addr | $request
```
Example:
```
2025-05-04T10:23:01+00:00 | 200 | 0.003s | 172.18.0.2:3000 | GET /healthz HTTP/1.1
```
